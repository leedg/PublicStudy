diag_arch — 전체 아키텍처

Client → NetworkEngine → BaseNetworkEngine → Windows/Linux/macOS → I/O Provider 계층 구조. 현재 영어 위주, 초보자용 설명 없음.

2

diag_seq — 연결·패킷 흐름 시퀀스

TCP connect → Session 생성 → Recv/Send 흐름. 화살표 라벨 9px로 너무 작고, 모두 영어.

3

diag_async_1 — 스레드 디스패치

IO Worker → KeyedDispatcher → Logic Worker → DB Layer. 이미 한국어 설명 일부 있음.

4

diag_async_2/3 — Keyed·ExecQueue 상세

아직 파악 중. 내용 확인 필요.

5

diag_db — DB 처리 상세

DBTaskQueue · WAL · IDatabase 계층. 한국어 많고 잘 구성됨. WAL 개념 설명 부족.
