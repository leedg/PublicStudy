공통 디자인 시스템

추가 ①
왼쪽 단계 번호 라벨
①②③ 번호로 흐름 순서를 명시. 독자가 "지금 어느 단계인지" 바로 파악.
추가 ②
박스 내 Korean subtitle
기술 용어 아래 작은 글씨로 "이게 하는 일" 한 줄 설명. 예: WAL → 충돌 시 복구용 임시 기록
추가 ③
비유 말풍선 (노란 박스)
복잡한 개념에 실생활 비유 삽입. 예: DBTaskQueue → "주방 주문 대기열"
추가 ④
하단 "핵심 포인트" 요약
각 다이어그램 맨 아래에 "이 그림에서 기억할 것 1가지" 초록 박스 추가.
변경 ⑤
화살표 라벨 폰트 9px → 12px
diag_seq의 모든 화살표 라벨 확대. 영어 라벨에 한국어 병기.
유지
기존 컬러 시스템 · 레이아웃
네이비/그린/오렌지 색상 코딩, 전체 구조는 최대한 유지. draw.io에서 계속 편집 가능.

다이어그램별 주요 변경사항

파일 현재 문제 변경 내용 추가될 비유
diag_arch 계층 설명 없음, 영어 위주 각 레이어에 "역할 설명" 텍스트 추가
Platform 박스에 Korean subtitle		 "건물 층 구조처럼, 위층은 아래층에게 일을 시킨다"
diag_seq 라벨 9px, 모두 영어, 참여자 5개로 복잡 ①②③ 번호 라벨, 화살표 한국어 병기
각 단계에 "무슨 일이 일어나는가" 노트		 "전화 연결처럼: 수락→등록→대기→통화→끊기"
diag_async_1 "completion event", "key-affinity" 영어 영어 용어 옆에 소괄호로 한국어 추가
DB Layer 박스 타이머 설명 보강		 "콜센터: I/O Worker=수신, Logic Worker=처리 담당자"
diag_async_2 TOCTOU 등 고급 용어, key%N 개념 설명 없음 "key % N = 나머지 연산" 시각적 예시 강화
기술 설명 박스 단순화		 "같은 손님(Session)은 항상 같은 직원(Worker)이 담당"
diag_async_3 CAS, false sharing 설명 없음 CAS 한 줄 설명 추가, Lock-Free 개념 박스 "Mutex=화장실 열쇠, Lock-Free=번호표 뽑기"
diag_db WAL 개념 설명 부족, Factory 패턴 불명확 WAL 비유 추가, Factory 역할 설명 박스 "WAL=영수증 먼저 발행, 실제 처리는 나중에"
작업 방식: draw.io XML을 직접 편집 → 6개 파일 각각 개선판 저장 → 완료 후 draw.io 앱에서 열어 확인/추가 편집 가능.
docx 텍스트: 각 섹션 앞에 "이 그림을 읽기 전에" 한 문단 안내문 추가 (복잡한 개념 사전 설명).
